Anchor node


An anchor node can be activated only by setting the activate field to true (no geometry picking supported yet).
The behavior then depends of the first item of url field:

  • M4M file and parameter is "self": only the Inline part containing this Anchor is replaced.
  • M4M file and parameter is NOT "self": the whole scene is replaced.
  • anything else: the external html browser is invoked with this url and the parameter (that can be a reference to a frame by instance).
    Note: MicroEmulator doesn't open web browser.

MFString url "" :
The first item of the url field can be either:

  • an M4M file (eg 'myScene.m4m') to jump to this file.

    Warning:
    To link a sub-scene, use M4M files in URL.
    Don't use WRL files ( cause problems during the final M4M file creation).

  • an HTTP url (eg 'http://www.google.com') to invoke the browser on this page.
  • an phone number with the "tel:" prefix (eg 'tel:336099034') to ask the user to make a call.

MFString parameter "" :
If this field contains "self", only the Inline part containing this Anchor is replaced.

SFBool activate FALSE :
Anchor will be activated when this is set to TRUE (or 1).

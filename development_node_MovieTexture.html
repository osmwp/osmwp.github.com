MovieTexture node

Interface


Description

Similar to ImageTexture with a video file. Used to map a video to any geometry in 3D context and to a Rectangle or Bitmap in a 2D context.


Fields

MFString url []:
Multiple item urls can be defined, but only the first one is loaded.
Items url can be of one of the following types:

  • Filename of an video (eg simple.mpg). The file will be included in the M4M file and immediately available on scene load.
  • An url to an image on the mobile filesystem (eg file://c/file.mpg).
  • An HTTP url (eg http://server/file.mpg) to download the file each time the scene is loaded.
  • A cache url followed by a source url (eg cache://file.mpg,http://server/file.mpg). File will be downloaded the first time from the source url and loaded from cache for later access.

SFTime startTime: Start time

SFTime stopTime: Stop time


Example

CONTROLS BY ROUTE

  • ROUTE mediaSensor.mediaCurrentTime => current playing position in the media
  • ROUTE mediaSensor.mediaDuration => total time of the media
  • ROUTE mediaSensor.status => status (opening, loading, lecture, stop...)

CONTROLS BY SCRIPT

  • mediaControl.pause => pause of player
  • mediaControl.volume => sound volume
  • mediaSensor.mediaCurrentTime => set playing position (if device is compatible)
